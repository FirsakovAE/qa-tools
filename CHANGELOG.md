## [2.0.0] - 2026-01-23
- Добавлена поддержка корпоративной версии приложения (не требующей установки);

## [2.1.0] - 2026-02-02
Общие изменения:
- Добавлена новая вкладка управления — Network.
- Проведён редизайн приложения с учётом нового расположения в iframe вместо popup. На экране теперь доступно больше полезной информации.
- Скорректирована вёрстка блоков JSON — теперь блок отображается на всю высоту приложения.
- Исправлено залипание вкладок при раннем открытии приложения, из-за которого ранее требовалась полная перезагрузка страницы для отображения вкладок props и pinia.

Props:
- Исправлена ошибка, при которой избранные props не находились после обновления страницы или повторного прохождения сценариев.
- Оптимизирована работа с props для длительных сессий без избыточной нагрузки на CPU, проведён рефакторинг.
- Переработана вкладка Props: оптимизирована работа с большим количеством объектов и поиск при наведении.
- Из вкладки Props удалён режим Props only.

Network:
- Перехват всех вызовов fetch и XMLHttpRequest (исключаются запросы типа OPTIONS и запросы, инициированные расширением).
- Отображение сводной информации по запросам: статус, метод, имя, время выполнения, размер.
- Подробный просмотр запроса с разделами: Параметры, Авторизация, Заголовки, Запрос, Ответ (отображаются только заголовки, доступные в рамках CORS-ограничений браузера).
- Приостановка и возобновление записи запросов (запись включена по умолчанию при первом открытии приложения).
- Breakpoint — остановка и модификация исходящего запроса перед отправкой.
- Mock Response — перехват заранее выбранного запроса с подменой ответа.
- Быстрое копирование в буфер обмена URL-адресов, токенов и ключевых элементов.
- Очистка всех записанных запросов одной кнопкой.
- Архитектура на основе событий без polling для минимального использования CPU.
- Автоматическое определение двоичного содержимого (отображаются только метаданные).

## [2.2.0] - 2026-02-24

### Общие изменения:
- Добавлена настройка **Display Mode** с возможностью переноса приложения во вкладку DevTools (отключение overlay-режима).
- Проведён рефакторинг вкладки **Options** — настройки разбиты на логические разделы.
- Изменена логика работы активных фильтров на вкладках **Props**, **Pinia Store** и **Network** — редактирование набора фильтров теперь доступно без перехода в настройки.
- Исправлена область нажатия (click area) в текстовом режиме отображения JSON.
- Большие JSON больше не обрезаются; для сохранения производительности при их отображении отключаются CSS-стили.
- Обновлена логика уведомления о новых версиях:
  - **Dismiss** переименована в **Later**;
  - **Download** переименована в **Preview** и ведёт в **Options → About**;
  - Добавлен блок с деталями версии и кнопками **Ignore** и **Download**.

### Props:

- Исправлена ошибка отсутствия подсветки UI props в режиме Standalone.

### Network:
- Добавлена поддержка form-data для чтения и подмены трафика.
- Добавлена возможность сохранить используемые файлы для повторного переиспользования при работе с Breakpoint с form-data.
- Добавлены режимы **Rewrite Breakpoint** и **Rewrite Mock**. При наличии существующего перехвата предлагается перезаписать правило вместо создания нового.
- Добавлена индикация активных **Breakpoint** и **Mock** для запроса; переход по счётчику открывает соответствующий раздел настроек.
- Добавлен поиск по статус-коду ответа.
- Расширены возможности **Breakpoint** — добавлен учёт HTTP-метода при перехвате.
- Исправлена ошибка, при которой инспектор не открывался при срабатывании Breakpoint.
- В контекстное меню запроса добавлены действия:
  - **Disable / Enable Breakpoint**;
  - **Disable / Enable Mock**;
  - **Delete Breakpoint / Mock**;
  - Пункты отображаются только при наличии соответствующего правила.

### Options:

#### General:
- Базовые настройки приложения (**Display Mode**, **Search Parameters**, **Auto Refresh**, **JSON Editor Mode**) перенесены в отдельный раздел.
- Управление файлом настроек (**Settings Management**) перенесено из шапки вкладки.

#### Network:
- **Search Settings** вынесены в отдельный раздел и стали независимыми.
- Скорректирована вёрстка таблиц **Breakpoints** и **Mocks** — данные больше не растягиваются.
- Добавлена возможность отключения и редактирования ранее созданных правил.

#### Props:
- Управление файлом настроек перенесено в раздел.
- **Component Blacklist** перенесён из общего блока настроек.
- **Favorite Components** перенесены из общего блока настроек.

#### Pinia Store:
- Управление файлом настроек перенесено в раздел.

#### About:
- Добавлены кнопки **Release notes** и **Check for updates** (доступна только в extension-версии).
- При наличии новой версии:
  - **Ignore** отключает повторные уведомления для текущей версии (сбрасывается при выходе более новой версии);
  - **Download** инициирует загрузку архива с приложением.

## [2.3.0] - 2026-03-19

### Настройки и хранение

#### Extension

* Исправлена работа избранного и чёрного списка при работе с **Props**.
* Изменён алгоритм хранения пользовательских настроек: приложенные файлы теперь сохраняются как ссылки на файлы устройства пользователя.

#### Standalone

* Корпоративная версия переведена на собственное хранилище.
* Настройки теперь сохраняются между запусками и доступны на разных сайтах без потери данных.

---

### Network

* Добавлена генерация **Postman-коллекций** на основе запросов, собранных во вкладке **Network**.

---

### Персонализация

В разделе **Options → General → Customize** появились настройки визуального оформления приложения.
Изменения применяются сразу после изменения и сохраняются в файл настроек.

Доступные параметры:

* **Dark / Light** — переключение цветовой схемы приложения.
* **Image** — выбор фонового изображения (**File** / **Link**) с поддержкой:
  * статичных изображений,
  * **GIF**,
  * **MP4** (живые обои).
* При выборе **File** изображение можно выбрать через **Browse** или из списка ранее сохранённых файлов.
* **Image Opacity** — прозрачность фонового изображения.
* **Blur** — степень размытия.
* **Noise Intensity** — интенсивность шумового эффекта.
* **Noise Opacity** — прозрачность шумового слоя.
* **Position (X / Y)** — положение изображения по осям.
* **Scale** — масштаб изображения (100–200%).

---

### Store

* Добавлен функционал избранного для **Pinia Store**.
* Элементы Store теперь можно сохранять в избранное.
* В разделе **Options** доступно редактирование и создание собственных избранных элементов.

---

### UI / UX

* Добавлена адаптивная раскладка интерфейса для небольших экранов.
* На широких экранах интерфейс работает в двухпанельном режиме: список и детали отображаются одновременно.
* При ширине около **1000px** интерфейс автоматически переключается в однопанельный режим: сначала отображается список, затем детали выбранного элемента.
* При однопанельном формате во вкладке Network изменяется кнопка Copy cURL на контекстное меню с доступом к Breakpoint / Mock / Copy cURL;
* Для обоих режимов добавлена логика вызова контекстоного меню ПКМ по шапке детализации запроса Network.
* Добавлена поддержка контекстного меню (**ПКМ**).
* Добавлено меню быстрых действий для **Props**.
* Добавлена поддержка клавиши **Esc** для закрытия активных панелей и диалогов.
* Добавлена поддержка поиска **Ctrl+F** в режиме вкладки **DevTools**.
* Добавлена возможность обновления открытого компонента — в режиме чтения отображается соответствующая кнопка.
* Изменён дизайн шеврона для открытия iframe.
* Добавлены shortcut-переходы к спискам избранного при нажатии на счётчик избранных элементов **Props** и **Store**.
* Добавлены настройки отображения колонок для таблиц Props, Pinia и Network.
* Добавлены скелетоны вовремя загрузки таблиц Props и Pinia вместо текста Not found.

* Исправлена область нажатия для блока JSON при редактировании во время скролла.
