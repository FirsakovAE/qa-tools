## [2.0.0] - 2026-01-23
- Добавлена поддержка корпоративной версии приложения (не требующей установки);
- Добавлена система ослеживания версий и уведомлений о доступности новых обновлений;
- Проект переведен в формат iframe и больше не использует popup как точку входа
- Решена проблема мгновенной утечки памяти для статичных сайтов типа docusaurus или hugo;
- Проведен рефакторинг стилей и оформления продукта;

## [2.1.0] - 2026-02-07
Общие изменения:
- Добавлена новая вкладка управления — Network.
- Проведён редизайн приложения с учётом нового расположения в iframe вместо popup. На экране теперь доступно больше полезной информации.
- Скорректирована вёрстка блоков JSON — теперь блок отображается на всю высоту приложения.
- Исправлено залипание вкладок при раннем открытии приложения, из-за которого ранее требовалась полная перезагрузка страницы для отображения вкладок props и pinia.

Props:
- Исправлена ошибка, при которой избранные props не находились после обновления страницы или повторного прохождения сценариев.
- Оптимизирована работа с props для длительных сессий без избыточной нагрузки на CPU, проведён рефакторинг.
- Переработана вкладка Props: оптимизирована работа с большим количеством объектов и поиск при наведении.
- Из вкладки Props удалён режим Props only.

Network:
- Перехват всех вызовов fetch и XMLHttpRequest (исключаются запросы типа OPTIONS и запросы, инициированные расширением).
- Отображение сводной информации по запросам: статус, метод, имя, время выполнения, размер.
- Подробный просмотр запроса с разделами: Параметры, Авторизация, Заголовки, Запрос, Ответ (отображаются только заголовки, доступные в рамках CORS-ограничений браузера).
- Приостановка и возобновление записи запросов (запись включена по умолчанию при первом открытии приложения).
- Breakpoint — остановка и модификация исходящего запроса перед отправкой.
- Mock Response — перехват заранее выбранного запроса с подменой ответа.
- Быстрое копирование в буфер обмена URL-адресов, токенов и ключевых элементов.
- Очистка всех записанных запросов одной кнопкой.
- Архитектура на основе событий без polling для минимального использования CPU.
- Автоматическое определение двоичного содержимого (отображаются только метаданные).
