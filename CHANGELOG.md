# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),

## [Unreleased]
### Added
- Режим расширения
- ** Вкладка "Сеть" **: Новая вкладка для перехвата и проверки сетевых запросов
 - Перехватывает все вызовы fetch и XMLHttpRequest (исключает ОПЦИИ и запросы на расширение)
 - Архитектура на основе событий без опроса для минимального использования процессора
 - Отображает сводную информацию о запросе: Статус, метод, название, время, размер
 - Подробный просмотр с разделами: Параметры, Авторизация, Заголовки, Запрос, Ответ
 - Функция приостановки / возобновления записи (запись включена по умолчанию)
 - Кнопка очистки всех записанных запросов
 - Буфер FIFO с настраиваемым максимальным количеством записей (по умолчанию: 500)
 - Усечение основного текста с настраиваемым максимальным размером (по умолчанию: 100 КБ)
 - Обнаружение двоичного содержимого (отображает только метаданные)
 - Функция копирования в буфер обмена URL-адресов, токенов и основных элементов

## [2.0.0] - 2026-01-23
- Добавлена поддержка корпоративной версии приложения (не требующей установки);
- Добавлена система ослеживания версий и уведомлений о доступности новых обновлений;
- Проект переведен в формат iframe и больше не использует popup как точку входа
- Решена проблема мгновенной утечки памяти для статичных сайтов типа docusaurus или hugo;
- Проведен рефакторинг стилей и оформления продукта;

## [2.1.0] - 2026-02-07
- Исправлена ошибка при которой избранные props не находились при обновлении страницы или перепрохождении сценариев;
- Исправлено залипание табов при раннем открытии приложения, из за чего приходилось полностью перезагружать страницу для появления вкладок props и pinia;
- Скорректирована верстка блоков JSON, теперь блок отображается на всю высоту приложения;
- Работа с пропсами оптимизирована для более длительной работы с приложением без загрузки CPU, проведены работы по рефакторингу;
