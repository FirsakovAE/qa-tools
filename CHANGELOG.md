* Добавлены режимы **Rewrite Breakpoint** и **Rewrite Mock**. При наличии ранее установленного перехвата в выпадающем меню теперь предлагается перезаписать существующее правило вместо создания нового.
* Проведён редизайн вкладки **Options**.
* На вкладках **Props**, **Pinia Store** и **Network** изменена логика работы тегов активных фильтров — теперь можно редактировать набор фильтров без перехода в настройки.
* Для текстового режима JSON исправлена область нажатия (click area).

### Network

* Добавлена индикация активных **Breakpoint** и **Mock** для запроса. Нажатие на счётчик перенаправляет в соответствующий раздел настроек.
* Добавлен поиск по статус-коду ответа.
* Переименованы теги поиска для повышения прозрачности и предсказуемости фильтрации.
* В контекстное меню запроса добавлены действия:

  * **Disable / Enable Breakpoint**
  * **Delete Breakpoint**
  * **Disable / Enable Mock**
  * **Delete Mock**
    Пункты отображаются только при наличии соответствующего правила у выбранного запроса.

### Options → About

* Добавлены кнопки **[ Release notes ]** и **[ Check for updates ]**.
* **Release notes**:

  * Для extension-версии выполняется `GET /releases/tags/v{version}`.
  * Для standalone-версии — `GET /releases/latest`.
  * В блоке деталей отображается содержимое поля `body` ответа GitHub API.
* **Check for updates** (доступна только в extension-версии):

  * Выполняется `GET /releases/latest`.
  * Добавлен loading state.
  * Кнопка блокируется на время выполнения запроса.
  * Обработаны ответы:

    * `304 Not Modified` (при использовании ETag).
    * Ограничение GitHub API (rate limit 60 запросов в час без токена).

#### Если доступна новая версия

В блоке деталей справа отображается:

* Заголовок: **Update Available**
* Подзаголовок: **Download new version {version}**
* Описание из поля `body`.
* Кнопки:

  * **Ignore** — отключает повторное отображение `CustomUpdateNotification.vue` для текущей версии. При изменении версии флаг игнорирования сбрасывается.
  * **Download** — инициирует загрузку по ссылке `assets[0].browser_download_url`.

## [2.1.0] - 2026-02-02
Общие изменения:
- Добавлена новая вкладка управления — Network.
- Проведён редизайн приложения с учётом нового расположения в iframe вместо popup. На экране теперь доступно больше полезной информации.
- Скорректирована вёрстка блоков JSON — теперь блок отображается на всю высоту приложения.
- Исправлено залипание вкладок при раннем открытии приложения, из-за которого ранее требовалась полная перезагрузка страницы для отображения вкладок props и pinia.

Props:
- Исправлена ошибка, при которой избранные props не находились после обновления страницы или повторного прохождения сценариев.
- Оптимизирована работа с props для длительных сессий без избыточной нагрузки на CPU, проведён рефакторинг.
- Переработана вкладка Props: оптимизирована работа с большим количеством объектов и поиск при наведении.
- Из вкладки Props удалён режим Props only.

Network:
- Перехват всех вызовов fetch и XMLHttpRequest (исключаются запросы типа OPTIONS и запросы, инициированные расширением).
- Отображение сводной информации по запросам: статус, метод, имя, время выполнения, размер.
- Подробный просмотр запроса с разделами: Параметры, Авторизация, Заголовки, Запрос, Ответ (отображаются только заголовки, доступные в рамках CORS-ограничений браузера).
- Приостановка и возобновление записи запросов (запись включена по умолчанию при первом открытии приложения).
- Breakpoint — остановка и модификация исходящего запроса перед отправкой.
- Mock Response — перехват заранее выбранного запроса с подменой ответа.
- Быстрое копирование в буфер обмена URL-адресов, токенов и ключевых элементов.
- Очистка всех записанных запросов одной кнопкой.
- Архитектура на основе событий без polling для минимального использования CPU.
- Автоматическое определение двоичного содержимого (отображаются только метаданные).
