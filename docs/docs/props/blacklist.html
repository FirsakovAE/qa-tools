Чёрный список компонентов (Props) ​

Чёрный список в контексте вкладки Props — это набор имён компонентов Vue, которые при сборе дерева не показываются в таблице инспектора. Фильтрация выполняется на стороне страницы, до тяжёлого чтения props: такие инстансы не попадают в список и не тратят ресурсы на сериализацию.

Общее описание вкладки Props: Основные возможности.

Не путайте с чёрным списком сайтов для автозапуска оверлея (отдельная логика по URL страницы в настройках General / Auto Run). Здесь речь только об именах компонентов.

Где настраивать ​

Секция Component Blacklist находится в Options инспектора рядом с настройками поиска по Props и режимом Inspect (в приложении это вкладка настроек, блок Props). Там же задаётся список правил, их включение и исключения.

Дополнительно имя можно добавить в чёрный список из таблицы Props через действие вроде «игнорировать по имени» у строки (точное имя текущего компонента попадает в активный список, если его ещё не было).

Активные и неактивные правила ​

Список разделён на два набора строк:

  • Активные (Blocked) — правило действует: совпавшие по имени компоненты скрыты из дерева.
  • Неактивные (Off / переведённые действием «разрешить») — строка хранится, но не блокирует. Её можно снова включить («заблокировать») без повторного ввода имени.

Удаление строки полностью убирает правило из обоих наборов.

Пока в активном списке нет ни одного правила, фильтрация по чёрному списку фактически выключена (строки только среди неактивных используются после следующего включения).

Как сопоставляется имя ​

Сравнение идёт с именем компонента так, как его видит инспектор (в т.ч. для безымянных экземпляров может использоваться запасной ярлык вроде Anonymous на стороне кода).

  • Если в правиле нет символов * и %, имя должно совпасть буквально (без учёта регистра): например правило MyButton скрывает только компоненты с таким именем.
  • Символы * и % в правиле трактуются как подстановка произвольного фрагмента (по смыслу «начинается / содержит / маска» в одном шаблоне, оформленном как одна строка с подстановками). Так можно задать, например, общий префикс для целого семейства компонентов.

Если имя компонента совпало с неактивным правилом‑исключением, оно не считается заблокированным, даже если подходит под другое активное правило (исключение проверяется первым). Это удобно, чтобы временно «пропустить» узкий поднабор имён внутри широкой маски.

Когда применяется ​

При каждом запросе списка компонентов (Refresh, автообновление, первое открытие) на страницу передаётся актуальный чёрный список вместе с запросом. После изменения правил в настройках дерево Props обычно перезапрашивается, чтобы таблица сразу отражала новый набор.

Заблокированные компоненты исчезают из основного списка: их нельзя выбрать карточкой, пока правило активно (если только не снять блок или не ослабить шаблон).

Практические советы ​

  • Начинайте с точных имён из таблицы, если нужно убрать шум от одного известного компонента.
  • Широкие маски экономят время, но легко скрыть лишнее — используйте неактивные исключения или уточняйте шаблон.
  • Чёрный список не удаляет компонент из приложения Vue; он влияет только на отображение в Vue Inspector.
