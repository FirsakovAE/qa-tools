# Работа с Pinia Store

Раздел **Stores** предназначен для просмотра зарегистрированных в приложении **Pinia**‑сторов: имя (идентификатор), число полей **state** и **getters**, а после выбора строки — снимок **state** и **getters** с возможностью правок и записи обратно в приложение (в поддерживаемых случаях).

Подробнее про редактирование **state** и **getters**, а также про **избранное** по сторам:

- [Работа со State](/store/state)
- [Работа с Getters](/store/getters)
- [Избранное](/store/favorite)

---

## Когда доступны данные

Модуль Pinia во внедрённом сценарии подключается **после обнаружения Pinia** на странице (логика сходна с вкладкой **Props**: сначала должны быть Vue и активный инспектор). Если Pinia не установлена или ещё не инициализирована при открытии вкладки **Stores**, список будет пустым или появится сообщение об ошибке загрузки.

Панель перед запросом сводки проверяет доступность канала связи со страницей (если контекст недоступен, данные не запрашиваются).

---

## Что именно отображается

1. **Сводка по всем сторам** — лёгкий ответ без полной сериализации каждого значения: для каждого стора передаются внутренний `id`, человекочитаемое имя (**baseId**), счётчик ключей **state**, счётчик **getters** и метка времени обновления снимка на стороне страницы.
2. **Полные данные выбранного стора** — по отдельному запросу подгружаются сериализованные **state** и **getters** для карточки и редактирования.

Так отделяется **быстрый список** для таблицы от **тяжёлого чтения** одного стора по выбору пользователя.

---

## Таблица сторов

Колонки (видимость задаётся в настройках таблицы):

- **Name** — отображаемое имя стора (**baseId**);
- **State** — число ключей в state (если нет — прочерк);
- **Getters** — число getters (если нет — прочерк).

У строки доступны **звезда избранного** и контекстное меню. Избранное хранится в настройках инспектора; см. [Избранное](/store/favorite).

---

## Обновление данных

- Кнопка **Refresh** в панели **Stores** заново запрашивает сводку у страницы.
- В карточке стора есть **Refresh data** — перечитывает **state** / **getters** для открытого стора без перезагрузки всей таблицы.
- Если в **Options** включено **автообновление**, сводка запрашивается по интервалу из настроек; при скрытии панели интервал останавливается (как и на вкладке Props).

Данные в карточке — **снимок на момент запроса**: изменения в приложении извне не подсвечиваются автоматически, пока вы не обновите данные.

## Поиск

Поле поиска и переключатель **Search by** работают так:

| Режим | Назначение |
| ----- | ---------- |
| **Name** | фильтр по имени стора (локально в панели) |
| **Key** | совпадение по **именам** полей в **state** и **getters** (запрос уходит на страницу) |
| **Value** | совпадение по **значениям** в **state** и **getters** (так же на странице) |

Для **Key** и **Value** действует минимальная длина запроса (по умолчанию **2** символа; настраивается в параметрах поиска инспектора). Точное совпадение — если весь запрос обёрнут в **двойные кавычки** `"..."`.

На стороне поиска по содержимому сторов действует **лимит числа результатов** (порядка сотни совпадений), чтобы не перегружать страницу при широком запросе.

---

## Карточка стора

После выбора строки открывается панель с вкладками **State** и **Getters** (если соответствующие секции непусты в сводке), редактором JSON (текст или дерево — по настройкам инспектора), кнопками **редактирования / сохранения** и возвратом к списку. Детали правок и ограничений — на страницах [State](/store/state) и [Getters](/store/getters).

---

## Как устроен обмен (для прозрачности)

Панель не читает Pinia напрямую из памяти процесса расширения: она шлёт сообщение в контекст страницы, где уже есть доступ к `pinia._s` и инстансам сторов.

Упрощённо загрузка списка выглядит так:

```js
// Иллюстрация: запрос сводки — только id стора, имя и счётчики ключей, без полного state
const response = await bridge.send({ type: 'PINIA_GET_STORES_SUMMARY' })
// response.summary — объект «storeId → { id, baseId, stateKeys, getterKeys, … }»

// Полный снимок одного стора — отдельным сообщением:
const detail = await bridge.send({
  type: 'PINIA_GET_STORE_STATE',
  storeId: 'my-store-id',
})
// detail.state, detail.getters — для отображения и операций PATCH / REPLACE (при поддержке стором)
```

Поиск по ключам и значениям (`PINIA_SEARCH`) выполняется на странице и возвращает только список идентификаторов подходящих сторов.

## Ограничения и ожидаемое поведение

- Поддерживается **Pinia** в типичной связке с Vue; кастомные регистрации сторов или крайние сценарии могут отображаться неполно.
- Сериализация **state** / **getters** для JSON ограничена здравыми пределами глубины и размера: циклические структуры, функции, спецобъекты могут быть усечены или представлены не так, как в «сыром» runtime.
- Права на запись (**replace state**, **patch getters**) зависят от того, как объявлен стор (например, доступ к внутренним полям через `$patch` и т.д.) — при ошибке сохранения смотрите сообщение в интерфейсе или консоль.
