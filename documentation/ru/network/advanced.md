# Расширенный режим захвата (Advanced mode)

В разделе **Options → Network** можно выбрать **режим захвата** сетевого обмена:

| Режим | В интерфейсе | Суть |
| ----- | -------------- | ---- |
| **Classic mode** | «классический» снимок | Те же ограничения, что у перехвата на странице: видны заголовки и тело в объёме, доступном скрипту страницы (в т.ч. правила **CORS** для чужого origin). Подробнее см. [Работа с запросами](./general.md#ограничения-отображения). |
| **Advanced mode** | расширенный захват | Расширение запрашивает доступ к **`webRequest`** и дополняет карточку запроса **полным набором заголовков** запроса и ответа так, как их видит браузер на уровне обмена, а не только «разрешённый странице» список. |

Включение **Advanced mode** и сопутствующие действия (закрепление заголовков, **Header links**, редактирование правил со **Extract regex** и **Transform**) доступны **только в расширении** для Chrome / Edge.  
**В Standalone** по-прежнему используется сценарий без расширенного захвата: полный набор заголовков через **`webRequest`**, описанный ниже, там **недоступен**.

---

## Что даёт Advanced mode во вкладке Network

- **Заголовки запроса и ответа** в детализации запроса отражают расширенные данные (в том числе служебные и не попавшие в `Access-Control-Expose-Headers`).
- **Закрепление строк заголовков (pin)** — для имени заголовка можно закрепить строку **вверху списка** отдельно для блока **Request** и отдельно для **Response**. Закрепления выполняются из **меню строки** заголовка (три точки) при активном Advanced mode.
- **Header links** — правило «для этого заголовка на этом хосте открыть внешний URL», в шаблоне подставляется подготовленное значение вместо плейсхолдера `{value}`.

Управление списком правил **Header links** и таблицей **Pinned headers** в **Options → Network** показывается **только при включённом Advanced mode**, чтобы не загружать настройки в Classic mode.

## Header links

Правило задаёт:

- **Header** — имя HTTP-заголовка (сопоставление **без учёта регистра**);
- **Host** — хост URL запроса (нормализуется, без схемы и пути, например `api.example.com`);
- **Link** — шаблон URL, где фрагмент **`{value}`** заменяется на **итоговую строку** после этапов извлечения и преобразования (см. ниже).

Создание и правка правил доступны из **меню строки заголовка** во вкладке **Network** (Advanced) и из **Options** при выбранной группе правил для заголовка.

Подстановка **`{value}`** выполняется так:

1. Берётся **сырое** значение заголовка из карточки запроса.
2. Необязательно применяется **Extract regex** (если поле задано).
3. Необязательно применяется цепочка **Transform** (если поле задано).
4. Результат подставляется во все вхождения **`{value}`** в шаблоне ссылки.

Поля **Extract regex** и **Transform** **независимы**: можно задать только regex, только transform или оба.

---

## Extract regex

- Строка трактуется как **паттерн** для конструктора **`new RegExp(pattern)`** (без отдельного аргумента с флагами; при необходимости флаги включаются в литерал регулярного выражения в самой строке, если синтаксис это допускает).
- Выполняется **`String.prototype.match`** для **сырого** значения заголовка.
- Если есть **первая захватывающая группа** и она участвует в совпадении, в дальнейшую обработку идёт **она**.
- Если групп нет, используется **всё совпадение** целиком.
- Если паттерн **некорректен** или **совпадения нет**, для следующего шага используется **исходное** значение заголовка (как до regex).

### Примеры паттернов

```regex
([a-f0-9-]+)
```

```regex
(\d+)
```

```regex
https?:\/\/([^\/]+)
```

## Transform

Строка **Transform** описывает цепочку шагов, разделённых символом **`|`** (вертикальная черта). Шаги выполняются **слева направо**, каждый получает результат предыдущего.

Каждый шаг — вызов «функции» в скобках, по смыслу близко к подстановке в коде:

| Функция | Назначение | Пример |
| ------- | ---------- | ------ |
| **`trim()`** | Убрать пробелы в начале и конце | `"  abc  "` → `"abc"` |
| **`lowercase()`** | Нижний регистр | `ABC` → `abc` |
| **`uppercase()`** | Верхний регистр | `abc` → `ABC` |
| **`replace("a","b")`** | Заменить все вхождения подстроки **a** на **b** | `replace("-", "")` |
| **`substring(start,end)`** | Срез как в **`String.prototype.slice(start, end)`** | `substring(0, 8)` |
| **`removeNonDigits()`** | Оставить только цифры | `+54 (11) 5555` → `54115555` |
| **`prefix("x")`** | Префикс | `prefix("id-")` |
| **`suffix("x")`** | Суффикс | `suffix("-prod")` |

Аргументы в **`replace`**, **`prefix`**, **`suffix`** задаются в **двойных** или **одинарных** кавычках. Внутри строки символ **`\\`** экранирует следующий символ.

Неизвестное имя «функции» или ошибка в числе аргументов для **substring** / **replace** приводит к тому, что **этот шаг пропускается** (значение на входе следующего шага не меняется из-за этого шага).

### Плейсхолдер и подсказка

Типовой пример цепочки в поле ввода:

```text
replace("-", "") | lowercase()
```

Короткие примеры для ориентации: **`trim()`**, **`lowercase()`**, **`replace("-", "")`**.

## Сквозные примеры

### Нормализация UUID

**Extract regex:**

```regex
([a-f0-9-]{36})
```

**Transform:**

```text
replace("-", "")
```

**Результат для подстановки в `{value}`:** шестнадцатеричная строка без дефисов, например `fbdb0effcaf44d4d86652a71f5556274`.

### Извлечь числовой идентификатор

**Extract regex:**

```regex
(\d+)
```

**Transform:**

```text
trim()
```

### Весь заголовок в нижний регистр

**Extract regex:**

```regex
(.+)
```

**Transform:**

```text
trim() | lowercase()
```

## Краткая памятка

- **Advanced mode** расширяет заголовки за счёт **`webRequest`**; **Standalone** этого режима не предоставляет.
- **Classic mode** — прежняя модель «видимости» заголовков для страницы.
- **Pin** — отдельно для списков **Request** / **Response**.
- **Header links:** шаблон с **`{value}`**, опционально **Extract regex**, затем **Transform** (цепочка через `|`).
